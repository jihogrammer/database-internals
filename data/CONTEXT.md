#### 1️⃣ **섹션 1) 개요 - 책의 핵심 추가**

**책의 내용:**
> "데이터를 효율적으로 디스크에 저장하려면 컴팩트하고 직렬화와 역직렬화가 쉬운 포맷으로 인코딩해야 한다."

**개선 전:**
- 파일 포맷의 4대 특성만 나열
- 컴팩트함, 직렬화/역직렬화 언급 부족

**개선 후:**
- 컴팩트함, 직렬화/역직렬화 용이성 강조
- 메모리와 디스크 접근 방식 차이 설명
  - 메모리: malloc()/free() 사용 가능
  - 디스크: read()/write() 함수만 사용
- read/write 함수의 제약 설명

**수정 라인:**
```markdown
**파일 포맷의 핵심 특성:**
- **컴팩트함**: 최소한의 디스크 공간 사용
- **직렬화/역직렬화 용이성**: 디스크 쓰기(write)와 읽기(read) 함수로 빠르게 변환 가능

**메모리와 디스크의 접근 방식 차이:**
디스크 접근: read()/write() 함수만 사용
           → 순차 I/O만 가능
           → 포인터 사용 불가능
           → 정확한 바이너리 레이아웃 필수
```
